# Oolio Onboard — v2 Deployment Guide

This update adds: **Oolio Brain** (admin-only knowledge store), **ChatGPT-style chat history**, **streaming responses**, **follow-up suggestions + "Got what I needed" button**, **owner role for Olivia**, **PWA install support**, and **62 pre-seeded Brain entries**.

## ⚠️ DEPLOY IN THIS ORDER

### Step 1 — Run the SQL migration (REQUIRED, do this FIRST)

This is **safe and additive** — it does NOT touch your existing users, knowledge, documents, or chat data. It only adds new tables and the owner role.

1. Go to Supabase → **SQL Editor** → **New query**
2. Open `supabase/migration-v2.sql` from this project
3. Copy/paste the **entire file** into the SQL editor
4. Click **Run**
5. Verify with:
   ```sql
   select email, role from public.profiles where role = 'owner';   -- should show olivia.mayes@oolio.com
   select count(*) from public.brain;                                -- should be 62
   select count(*) from public.chat_sessions;                        -- should be 0 (until first chat)
   ```

### Step 2 — Push the code to GitHub

```bash
git add .
git commit -m "v2: Oolio Brain, chat history, streaming, owner role"
git push
```

Vercel auto-deploys. No env var changes needed (Haiku uses your existing `ANTHROPIC_API_KEY`).

### Step 3 — Test the rollout

After deploy, verify:

1. **You're the owner.** Sign in → header shows "👑 owner" next to your name. Admin tab shows "👑 OWNER" badge.
2. **Brain works.** Admin → Oolio Brain → search for "pricing" or "OolioPay" → 62 entries pre-seeded.
3. **Chat history works.** Send a question → chat menu icon (top-left) → see your conversation in the sidebar.
4. **Streaming works.** Ask a question → response appears word-by-word.
5. **Follow-ups work.** After an answer, see 2 suggested follow-ups + green "Got what I needed" button.
6. **Owner view works.** Open sidebar → tick "Show all users' chats" → see everyone's chats (for coaching).
7. **PWA install.** On phone Safari/Chrome → Share → "Add to Home Screen" → icon installs.

## Troubleshooting

**"Brain tab empty"** — Run the SQL migration (Step 1). Verify with `select count(*) from public.brain` → should be 62.

**"I'm not the owner"** — The migration sets owner by email. Re-run:
```sql
update public.profiles set role = 'owner' where email = 'olivia.mayes@oolio.com';
```

**"Chat history sidebar is empty"** — Send a new chat first. The `chat_sessions` table is empty until the first conversation. Old chats from before this update won't appear in the sidebar (they were never linked to a session).

**"Streaming not working"** — Some corporate proxies buffer SSE. Falls back to non-streaming gracefully. Check Vercel function logs.

**"Got what I needed not appearing"** — It only shows after the AI completes its response. If follow-ups failed to generate (Haiku timeout), the button won't show. Refresh and try again.

**Old "Knowledge" data** — Still in the `knowledge` table (untouched). If you want to migrate any approved entries into Brain manually:
```sql
insert into public.brain (title, content, source_url)
select topic, content, source_url from public.knowledge where approved = true;
```
